## ◻ Test Tier C self-hosted runner deploy

Needs: GitHub admin + the Mac mini.

1. Register a self-hosted GitHub Actions runner on the box
   (GitHub → Settings → Actions → Runners — prefer an **ephemeral** runner).
2. Set the repo variable `SELF_HOSTED_DEPLOY = true`
   (Settings → Secrets and variables → Actions → Variables).
3. Push a `v*` tag → confirm `.github/workflows/deploy.yml` runs `make deploy`
   on the box.
4. Also confirm the workflow is **skipped** when the variable is unset.

## ◻ Full `make setup` run against a real domain

Needs: a Cloudflare domain + API/tunnel token.

1. Quick mode smoke test (brief public `*.trycloudflare.com` exposure —
   tear down right after): `SETUP_MODE=quick SETUP_YES=1 make setup`.
2. Guided or automated mode against the real domain → live
   `https://app.<domain>` with seeded admin and green
   `make tunnel-verify`.
3. Then exercise CD: set `APP_IMAGE` in `.env`, run `make deploy`, and roll
   back by re-pinning `APP_TAG`.
4. On the Mac mini, finish with `make autostart` + auto-login + pmset for
   boot persistence (docs/self-hosting.md → "Running on a Mac mini").
